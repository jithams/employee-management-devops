# Phase 1 - Linux Administration Report

## Objective

To understand and perform fundamental Linux system administration tasks on an AWS EC2 instance running Amazon Linux 2023.

## Environment

- AWS EC2
- Amazon Linux 2023
- Git
- GitHub

## Tasks Performed

### 1. Linux Installation and Configuration
- Launched an EC2 instance.
- Connected securely using SSH.
- Verified system information.

### 2. User and Group Management
- Created users.
- Created groups.
- Assigned users to groups.
- Verified user and group information.

### 3. File Permissions and ACL
- Modified file permissions.
- Changed file ownership.
- Configured Access Control Lists (ACL).

### 4. Package Management
- Installed required packages using DNF.
- Verified package installation.

### 5. Service Management
- Started services.
- Enabled services at boot.
- Checked service status.

### 6. Storage Management
- Created a new EBS volume.
- Attached the volume to the EC2 instance.
- Partitioned the disk.
- Formatted the partition using XFS.
- Mounted the filesystem.
- Configured persistent mounting using /etc/fstab.

### 7. SSH Configuration
- Verified SSH service.
- Tested SSH connectivity.
- Reviewed SSH configuration.

### 8. Firewall Configuration
- Installed and configured firewalld.
- Allowed HTTP, HTTPS and SSH services.
- Verified firewall rules.

### 9. Shell Scripting
- Developed a backup script.
- Automated backup of the project directory.

### 10. Log Management and Troubleshooting
- Viewed system logs.
- Monitored running services.
- Checked memory and disk usage.
- Performed basic troubleshooting.

## Conclusion

Successfully completed the Linux Administration phase by configuring users, storage, services, networking, security and automation on an AWS EC2 instance.
