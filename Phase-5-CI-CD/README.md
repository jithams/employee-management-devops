# Phase 5 – CI/CD Implementation

## Objective

The objective of this phase is to implement a Continuous Integration and Continuous Deployment (CI/CD) pipeline using Jenkins. The pipeline automates the build, testing, deployment, and rollback process for the Employee Management application.

---

## Tools Used

- Jenkins
- Git
- GitHub
- Apache HTTP Server
- Amazon Linux 2023
- AWS EC2
- Shell Scripting

---

## Tasks Completed

### Step 1 – Jenkins Setup

- Launch EC2 Instance
- Install Java
- Install Jenkins
- Unlock Jenkins
- Create Admin User
- Install Required Plugins
- Configure Jenkins

---

### Step 2 – Automated Build Process

- Connected GitHub Repository
- Created Jenkins Freestyle Job
- Configured Source Code Management
- Executed Automated Build
- Verified Successful Build

---

### Step 3 – Automated Testing

- Configured Test Stage
- Executed Test Commands
- Verified Test Results

---

### Step 4 – Automated Deployment Pipeline

- Configured Deployment Stage
- Deployed Application Automatically
- Verified Deployment

---

### Step 5 – Rollback Strategy

- Simulated Deployment Failure
- Restored Previous Working Version
- Verified Successful Rollback

---

## Result

Successfully implemented a CI/CD pipeline using Jenkins to automate application build, testing, deployment, and rollback operations.
