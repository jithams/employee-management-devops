# Phase 2 - Advanced Linux Administration (RHCE)

## Objective

The objective of this phase was to automate Linux server administration using Ansible. This included configuring Ansible, creating playbooks, implementing role-based automation, managing server configurations, deploying an Employee Management Portal, and provisioning servers automatically.

---

## Environment

- Operating System: Amazon Linux 2023
- Platform: AWS EC2
- Control Node: capstone-server
- Managed Node: employee-node
- Automation Tool: Ansible
- Web Server: Apache (httpd)

---

## Tasks Completed

### Step 1 - Ansible Installation and Configuration
- Installed Ansible
- Configured inventory
- Configured passwordless SSH
- Verified connectivity using Ansible Ping

### Step 2 - Infrastructure Automation
- Created Ansible playbooks
- Installed Apache automatically
- Managed users and services
- Verified successful playbook execution

### Step 3 - Role-Based Automation
- Created an Apache role
- Organized tasks, handlers, files, and variables
- Executed the role using a playbook

### Step 4 - Configuration Management
- Managed website content using Ansible
- Updated the web page automatically
- Verified configuration changes

### Step 5 - Automated Application Deployment
- Created an Employee Management Portal
- Deployed the application automatically using Ansible
- Verified successful deployment in the browser

### Step 6 - Server Provisioning
- Created a provisioning playbook
- Automated complete server setup
- Verified Apache service and application deployment

Successfully completed Phase 2 by automating Linux server administration using Ansible. The Employee Management Portal was deployed successfully, demonstrating infrastructure automation, configuration management, and server provisioning using Ansible playbooks and roles.
